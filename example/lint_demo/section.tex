% Top-level comment with DAVIS-346 inside — paperterm should ignore this line entirely.

We benchmark against the DAVIS346 reference sensor.  Some passages
in older drafts wrote DAVIS-346, Davis346, or even davis346, none
of which match the canonical capitalisation.

The simulator we compare with is DVS-Voltmeter, but a few sentences
elsewhere call it DVS Voltmeter or run the words together as
DVSVoltmeter.

The upstream event-statistics diagnostic is what we report in
Table~\ref{tab:summary}.  Earlier sections sometimes refer to it as
the upstream diagnostic, the upstream event-statistics scorecard,
or even the upstream event-statistics table; all three should be
flagged.

We compute per-pixel EMD across all variants.  Inconsistent
phrasings include spatial per-pixel EMD, per-pixel count EMD, and
pixel-level EMD.  In free prose we should never see the form
spatial px-EMD either.

Per-pair K calibration is the recommended workflow.  Sloppy
shorthand ``per-pair calibration'' loses the K, and
per-camera-pair K calibration is needlessly verbose.

Inline math like $|\Delta p|$ and DAVIS-346-shaped tokens inside
\cite{DAVIS-346:2014, davis346-extended} or \ref{tab:DAVIS-346-summary}
should NOT trigger any warning.

\begin{verbatim}
DAVIS-346 in a verbatim block is also fine; paperterm skips it.
\end{verbatim}
